idOS — Quick MCQ Quiz

Five multiple-choice questions about idOS. Answers are included in the Answers section below.

  1. What is the primary goal of idOS?
    A) Create a new blockchain specifically for payments
    B) Enable portable, reusable digital identity and KYC across on-chain and off-chain systems
    C) Replace all banks with decentralized exchanges
    D) Provide hardware wallets for institutional investors
  2. Which feature best describes idOS’s approach to KYC?
    A) One-time KYC per application only
    B) Centralized storage of government IDs with open public access
    C) Reusable KYC credentials that can be consented and shared across services
    D) Anonymity-first model with no identity verification
  3. Which of these is a component commonly associated with idOS architecture?
    A) Storage Network for identity data and an Economy Network for governance/token flows
    B) Video streaming CDN only
    C) Desktop OS kernel
    D) Proprietary email service
  4. How does idOS aim to handle regulatory compliance and privacy?
    A) By making all identity data public on-chain
    B) Using consented, privacy-preserving credential sharing and compliance modules
    C) Requiring users to email regulators manually
    D) Removing KYC requirements entirely
  5. Which is a realistic use case for idOS?
    A) Neobanks reusing verified KYC credentials to onboard customers faster
    B) Printing physical ID cards only
    C) Acting as a videogame engine
    D) Serving as a social network replacement
Answers (click to reveal)
  1. Q1 — B
    idOS’s main objective is to enable portable, reusable digital identity (including KYC) across on-chain and off-chain ecosystems.
  2. Q2 — C
    idOS promotes reusable, consented KYC credentials so users and services can share verified identity information without repeating full KYC flows.
  3. Q3 — A
    idOS architecture typically references components like a Storage Network (identity data, credentials) and an Economy Network (tokens, incentives, governance).
  4. Q4 — B
    idOS balances privacy and compliance via consented credential sharing and compliance tooling rather than exposing raw identity data publicly.
  5. Q5 — A
    A common idOS use case: neobanks or financial services reusing verified credentials for faster, compliant onboarding.
